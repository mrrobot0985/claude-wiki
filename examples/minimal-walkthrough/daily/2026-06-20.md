# Daily Log: 2026-06-20

## Sessions

### Session (09:00)

**Context:** Exploring Astral's `uv` toolchain for Python packaging.

**Key Exchanges:**

- `uv` is a Rust-based replacement for `pip`, `virtualenv`, and `python -m venv`.
- `uv run script.py` automatically creates or reuses a `.venv` if `pyproject.toml` is present.
- `uvx tool-name` executes a one-off CLI tool without installing it into the project.

**Decisions Made:**

- Switch all personal Python projects to `uv` for dependency management.
- Replace `make` targets that call `python -m` with `uv run` equivalents.

**Lessons Learned:**

- `uv` caches wheels globally, so repeated installs across projects are nearly instant.
- Project mode in `uv` removes the need to manually activate virtual environments.
- `uvx` is ideal for linters and formatters used in CI pipelines.

**Action Items:**

- Update `Makefile` to use `uv run` instead of `python -m`.
- Evaluate whether `uv` reduces CI build times for Docker-based workflows.
